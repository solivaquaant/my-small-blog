# KSECRETS

- **Description:** We have a kubernetes cluster setup and flag is in the secrets. You think you can get it?

## 🔎 Solution
This challenge focuses on interacting with a Kubernetes cluster. 
After selecting "Launch instance", two items are provided:
1. The Kubernetes server endpoint
2. A configuration file `config.yaml` used to connect to the cluster

Kubernetes is managed via the `kubectl` CLI tool. 
To ensure it connects to the correct server, we need to update the endpoint inside `config.yaml` with the provided address:
```
sed -i 's|https://127.0.0.1:6443|https://green-hill.picoctf.net:62545|g' config.yaml
```

Since the server uses a TLS certificate that does not match the hostname, we must include the `--insecure-skip-tls-verify` flag to bypass certificate validation. 
In Kubernetes, resources are organized into namespaces. 
We can enumerate all secrets across all namespaces using the `-A` flag:
```
> kubectl --kubeconfig=config.yaml --insecure-skip-tls-verify get secrets -A
NAMESPACE     NAME                       TYPE                               DATA   AGE
kube-system   chart-values-traefik       helmcharts.helm.cattle.io/values   1      4m51s
kube-system   chart-values-traefik-crd   helmcharts.helm.cattle.io/values   0      4m51s
kube-system   k3s-serving                kubernetes.io/tls                  2      4m52s
//highlight-next-line
picoctf       ctf-secret                 Opaque                             1      4m46s
```

From the output, we identify a secret named `ctf-secret` in the `picoctf` namespace.

Next, we retrieve the full details of this secret in YAML format to inspect its raw data. The `flag` field is Base64-encoded:
```
> kubectl --kubeconfig=config.yaml --insecure-skip-tls-verify get secret ctf-secret -n picoctf -o yaml
apiVersion: v1
data:
  flag: cGljb0NURntrczNjcjM3NV80MW43X3M0ZjNfMDE4YzRkOWJ9Cg==
kind: Secret
metadata:
  annotations:
    kubectl.kubernetes.io/last-applied-configuration: |
//highlight-next-line
      {"apiVersion":"v1","data":{"flag":"cGljb0NURntrczNjcjM3NV80MW43X3M0ZjNfMDE4YzRkOWJ9Cg=="},"kind":"Secret","metadata":{"annotations":{},"name":"ctf-secret","namespace":"picoctf"},"type":"Opaque"}
  creationTimestamp: "2026-03-17T08:38:23Z"
  name: ctf-secret
  namespace: picoctf
  resourceVersion: "387"
  uid: e8898118-ab0d-4093-8c24-f3c6ebe9cb42
type: Opaque
```

Decoding the Base64 value reveals the flag:
```
> echo "cGljb0NURntrczNjcjM3NV80MW43X3M0ZjNfMDE4YzRkOWJ9Cg" | base64 -d
picoCTF{ks3cr375_41n7_s4f3_018c4d9b}
```

## 🚩 Flag
> `picoCTF{ks3cr375_41n7_s4f3_018c4d9b}`
