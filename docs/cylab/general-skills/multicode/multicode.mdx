# MultiCode

- **Description:** We intercepted a suspiciously encoded message, but it's clearly hiding a flag. 
No encryption, just multiple layers of obfuscation. 
Can you peel back the layers and reveal the truth?

## 🔎 Solution
We are given the following encoded string:
```
> echo "NjM3NjcwNjI1MDQ3NTMyNTM3NDI2MTcyNjY2NzcyNzE1ZjcyNjE3MDMwNzE3NjYxNzQ1ZjM4NzE3MTMwMzM3MjczNzIyNTM3NDQ=" | base64 -d
637670625047532537426172666772715f72617030717661745f3871713033727372253744 =
```

At first glance, it looks like Base64, so I proceed with decoding it. The result gives the following string:

```
> echo "637670625047532537426172666772715f72617030717661745f3871713033727372253744" | xxd -r -p
cvpbPGS%7Barfgrq_rap0qvat_8qq03rsr%7D                                                                                                             
```

At this point, it's clear that we are very close to the flag. We can notice two special sequences: `%7B` and `%7D`, which correspond to `{` and `}` in URL encoding. 
The prefix `cvpbPGS` also looks similar to `picoCTF`, but appears to be encoded using some transformation.

A quick guess would be a ROT cipher. 
Trying ROT13 reveals the correct flag:
