Extract the content of `note.txt` located at inode `65692`:
```
icat -o 0001140736 disk.img 65692 > note.txt
```

The file content is simply a basic hint:
```
> cat note.txt                
The picoCTF flag format is 'picoCTF{}' where there is some leetspeak phrase in between the curly braces
```

Inside the `.git` directory, there are three Git object files:
```
+++++++ r/r 65695:      064ac3ab7afd9a95bc1224aa8b4cef23741fcc
+++++++ r/r 65698:      6ca660f488a4e4cdd92e7678fcfa3da478aee7
+++++++ r/r 65700:      7681bb38cf467cec328eec9707b240e3e74ced
```

Similarly, extract these files from the disk image:
```
> icat -o 0001140736 disk.img 65695 > obj1 
> icat -o 0001140736 disk.img 65698 > obj2
> icat -o 0001140736 disk.img 65700 > obj3
```

To view the raw content of Git objects, use `zlib-flate` since Git objects are compressed using `zlib`. 
The first two files do not contain anything particularly interesting, but the last one reveals the flag.
```
> zlib-flate -uncompress < obj3           
commit 232tree 186ca660f488a4e4cdd92e7678fcfa3da478aee7
author ctf-player <ctf-player@example.com> 1763542167 +0000
committer ctf-player <ctf-player@example.com> 1763542167 +0000

Wrap this phrase in the flag format: g17_1n_7h3_d15k_041217d8
```

## 🚩 Flag
> `picoCTF{g17_1n_7h3_d15k_041217d8}`
