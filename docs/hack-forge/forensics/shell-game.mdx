# Shell Game

- **Description:** During a security incident, a memory dump was captured from a workstation suspected of running malicious scripts. 
Your task is to analyze the memory and find evidence of the attacker's commands.
- **Difficulty:** Easy
- **Link:** [Shell Game](https://hackforge.com/challenges/shell-game)

## 🔎 Solution
In this challenge, we are provided with an ELF file. 
ELF (Executable and Linkable Format) is the standard file format for executables, object code, shared libraries, and core dumps on Unix-based systems:
```
> file shell-game.elf                                                                                           
shell-game.elf: ELF 64-bit LSB core file, x86-64, version 1 (SYSV)
```

Next, we inspect the ELF header. 
The important detail here is that the file type is identified as a core dump:
```
> readelf -h shell-game.elf 
ELF Header:
  Magic:   7f 45 4c 46 02 01 00 00 00 00 00 00 00 00 00 00 
  Class:                             ELF64
  Data:                              2's complement, little endian
  Version:                           0
  OS/ABI:                            UNIX - System V
  ABI Version:                       0
//highlight-next-line
  Type:                              CORE (Core file)
  Machine:                           Advanced Micro Devices X86-64
  Version:                           0x1
  Entry point address:               0x0
  Start of program headers:          64 (bytes into file)
  Start of section headers:          0 (bytes into file)
  Flags:                             0x0
  Size of this header:               64 (bytes)
  Size of program headers:           56 (bytes)
  Number of program headers:         21
  Size of section headers:           64 (bytes)
  Number of section headers:         0
  Section header string table index: 0
```

A core file is essentially a snapshot of system memory at the moment a program crashes or terminates unexpectedly. 
Because it captures memory, it may contain variable values, stack data, and even plaintext strings that were present at runtime.

Inspecting the file using `strings`, we notice several interesting keywords such as `VBOX`, `VBCORE`, and `VBCPU`. 
This suggests that the file is not a typical Linux ELF executable, but rather resembles a memory dump from a VirtualBox environment.

To dig further, we search for any PowerShell-related artifacts:
```
strings shell-game.elf | grep "powershell" 
```

The result reveals a suspicious command containing a Base64-encoded payload:
```
powershell.exe -enc VwByAGkAdABlAC0ASABvAHMAdAAgACIARgBMAEEARwBfAE0AQQBSAEsARQBSAF8AUwBUAEEAUgBUAD4APgA+AEgAYQBjAGsARgBvAHIAZwBlAHsAUAAwAHcAMwByAFMAaAAzAGwAbABfAEgAMQBzAHQAMAByAHkAXwBIAHUAbgB0ADMAcgB9ADwAPAA8AEYATABBAEcAXwBNAEEAUgBLAEUAUgBfAEUATgBEACIAOwAgAFMAdABhAHIAdAAtAFMAbABlAGUAcAAgAC0AUwBlAGMAbwBuAGQAcwAgADMANgAwADAA
```

Decoding this payload reveals the following command, which also contains the flag:
```
Write-Host "FLAG_MARKER_START>>>HackForge{P0w3rSh3ll_H1st0ry_Hunt3r}<<<FLAG_MARKER_END"; Start-Sleep -Seconds 3600 
```

## 🚩 Flag
> `HackForge{P0w3rSh3ll_H1st0ry_Hunt3r}`
